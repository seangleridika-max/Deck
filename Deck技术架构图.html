📊 数据模型关系

👤 users（用户表）

  1. id — 唯一标识符 (nanoid)
  2. email — 邮箱（登录凭证）
  3. name — 显示名称
  4. password_hash — 密码哈希
  5. plan — free / pro
  6. created_at — 注册时间

📁 sessions（项目表）

  1. id — Session ID (20位)
  2. user_id → users.id
  3. app_name — "deck"
  4. title — 项目标题
  5. skill_id — 使用的技能
  6. status — active/completed

📄 sources（资料来源）

  1. id — 来源ID
  2. session_id → sessions.id
  3. type — url/pdf/video/text
  4. url — 来源地址
  5. content — 提取内容
  6. created_at — 导入时间

📊 assets（资产表）

  1. id — 资产ID
  2. session_id → sessions.id
  3. filename — 文件名
  4. type — report/slides/source
  5. r2_key — R2对象键
  6. size — 文件大小(bytes)

🔄 核心请求流程

📚 资料导入流程

  1. 用户提交URL/上传文件
  2. 前端发送至 Fabric Worker
  3. Worker 验证Token并存储元数据至D1
  4. 文件上传至Cloudflare R2
  5. Gemini Flash 提取文本内容
  6. 内容存入D1 sources表
  7. 返回来源ID与摘要至前端

🔬 深度研究流程

  1. 用户提问或选择分析框架
  2. 前端聚合所有sources内容
  3. 构建 Gemini 2.5 Pro 请求
  4. 流式返回AI分析结果
  5. 结果附带来源引用标注
  6. 研究内容存至D1 logs表
  7. 用户可继续追问或生成报告

📄 报告生成流程

  1. 用户选择报告模板/技能
  2. Gemini Pro 基于研究内容生成初稿
  3. 前端富文本编辑器加载内容
  4. 用户在线编辑与调整
  5. 用户点击「导出」
  6. 前端生成DOCX/PDF文件
  7. 文件上传至R2并返回下载链接

🎞️ 幻灯片生成流程

  1. 用户点击「生成PPT」
  2. Gemini Pro 将报告分解为幻灯片结构
  3. JSON Schema 约束幻灯片数据格式
  4. 前端渲染幻灯片预览
  5. 用户拖拽编辑布局与内容
  6. pptxgenjs 生成最终PPTX文件
  7. 文件上传R2并返回下载链接

🔌 主要 API 端点

方法 端点 说明 认证
POST /users 创建用户账户 无
POST /users/login 用户登录，返回Token 无
GET /users/:id 获取用户信息 Bearer
POST /sessions 创建新项目（Session） Bearer
GET /sessions?appName=deck 列举当前用户项目 Bearer
POST /sessions/:id/sources 添加资料来源 Bearer
GET /sessions/:id/sources 获取项目资料列表 Bearer
POST /sessions/:id/logs 追加研究日志/对话记录 Bearer
GET /sessions/:id/logs 获取研究历史记录 Bearer
POST /sessions/:id/assets 上传报告/幻灯片文件（ZIP） Bearer
GET /sessions/:id/assets 获取资产列表（含签名URL） Bearer
GET /apps/deck/storage/* 应用级持久化存储（技能配置等） Bearer
